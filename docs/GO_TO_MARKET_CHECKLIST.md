# Docit Go-To-Market Checklist

## Product trust

- Keep README focused on outcomes and quick start.
- Maintain clear semantic versions (v1.0.0, v1.0.1, ...).
- Add at least 3 integration tests for real generation scenarios.
- Document required binaries and known platform caveats.

## Distribution

- Ensure Packagist package is indexed and auto-updating from GitHub.
- Tag releases with concise release notes.
- Keep composer metadata updated (keywords, support links, homepage).

## Community reach

- Submit Laravel News pitch (see LARAVEL_NEWS_PITCH.md).
- Post launch thread on X and LinkedIn.
- Share a practical tutorial article with working API examples.
- Record a short demo video (30-60s) showing template -> generated output.

## Viral levers

- Publish 3 sample templates users can copy immediately.
- Provide one command quickstart script in docs.
- Create before/after benchmarks versus manual pipelines.
- Invite users to share use cases and feature requests publicly.

## Weekly growth loop

- Collect feedback from issues and discussions.
- Turn top user question into one doc improvement per week.
- Ship small release every 1-2 weeks.
- Repost improvements with short changelog snippets.
